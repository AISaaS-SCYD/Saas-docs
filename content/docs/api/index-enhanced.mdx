## Authentication

<Callout type="warn">
All API requests must include an `Authorization` header with a valid API key:

```
Authorization: Bearer YOUR_API_KEY
```
</Callout>

## Rate Limiting

<Callout type="info">
**Rate Limits**:
- Free tier: 100 requests/minute
- Pro tier: 1,000 requests/minute
- Enterprise: Custom limits

Rate limit information is included in response headers:
- `X-RateLimit-Limit`: Maximum requests per window
- `X-RateLimit-Remaining`: Remaining requests in current window
- `X-RateLimit-Reset`: Timestamp when the limit resets
</Callout>

## Error Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Resource doesn't exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

## Frequently Asked Questions

<Accordions>
  <Accordion title="How do I paginate through large result sets?">
Use the `page` and `per_page` query parameters. The response includes a `pagination` object with total count and current page information.

```javascript
const response = await fetch('https://api.example.com/v1/users?page=2&per_page=50');
```
  </Accordion>

  <Accordion title="What's the maximum page size?">
The maximum `per_page` value is 100. Requesting more than 100 items will be automatically capped at 100.
  </Accordion>

  <Accordion title="How do I filter results?">
Most list endpoints support filtering via query parameters. Check the specific endpoint documentation for available filters.
  </Accordion>

  <Accordion title="Can I use OAuth instead of API keys?">
Yes! We support OAuth 2.0 for user-delegated access. See the [Authentication Guide](/docs/guides/authentication) for details.
  </Accordion>
</Accordions>
