# Authentication

Learn how to securely authenticate your API requests.

<Callout type="warn">
**Security First**: Never expose your API credentials in client-side code, public repositories, or logs.
</Callout>

## Authentication Methods

We support two primary authentication methods:

### API Keys

API keys are the simplest way to authenticate. They're ideal for server-to-server communication.

**Pros:**
- Simple to implement
- No expiration
- Great for backend services

**Cons:**
- Must be stored securely
- No granular permissions
- Harder to revoke access

#### Using API Keys

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
```javascript title="api-key-auth.js"
const fetch = require('node-fetch');

async function makeRequest() {
  const response = await fetch('https://api.example.com/v1/users', {
    headers: {
      'Authorization': `Bearer ${process.env.API_KEY}`,
      'Content-Type': 'application/json'
    }
  });

  return await response.json();
}
```

### OAuth 2.0

OAuth 2.0 is recommended for user-facing applications where you need to access user data on their behalf.

**Pros:**
- Secure user delegation
- Granular scopes
- Token expiration
- Easy to revoke

**Cons:**
- More complex to implement
- Requires user interaction

#### OAuth Flow

<Callout type="info">
We support the Authorization Code flow with PKCE for maximum security.
</Callout>

**Step 1: Redirect to Authorization URL**

```javascript title="oauth-step-1.js"
const authUrl = new URL('https://auth.example.com/oauth/authorize');
authUrl.searchParams.append('client_id', 'YOUR_CLIENT_ID');
authUrl.searchParams.append('redirect_uri', 'https://yourapp.com/callback');
authUrl.searchParams.append('response_type', 'code');
authUrl.searchParams.append('scope', 'read:users write:projects');

// Redirect user to authUrl
window.location.href = authUrl.toString();
```

**Step 2: Exchange Code for Token**

<Tabs items={['Node.js', 'Python']}>
  <Tab value="Node.js">
```javascript title="oauth-step-2.js"
async function exchangeCode(code) {
  const response = await fetch('https://auth.example.com/oauth/token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      grant_type: 'authorization_code',
      code: code,
      client_id: process.env.CLIENT_ID,
      client_secret: process.env.CLIENT_SECRET,
      redirect_uri: 'https://yourapp.com/callback'
    })
  });

  const { access_token, refresh_token } = await response.json();
  return { access_token, refresh_token };
}
```
  </Tab>

  <Tab value="Python">
```python title="oauth_step_2.py"
import requests
import os

def exchange_code(code):
    response = requests.post(
        'https://auth.example.com/oauth/token',
        json={
            'grant_type': 'authorization_code',
            'code': code,
            'client_id': os.getenv('CLIENT_ID'),
            'client_secret': os.getenv('CLIENT_SECRET'),
            'redirect_uri': 'https://yourapp.com/callback'
        }
    )

    data = response.json()
    return data['access_token'], data['refresh_token']
```
  </Tab>
</Tabs>

## Security Best Practices

<Callout type="error">
**Critical Security Rules**:
- Store credentials in environment variables or secrets managers
- Never commit credentials to version control
- Rotate API keys regularly
- Use HTTPS for all API requests
- Implement token refresh logic for OAuth
</Callout>

### Storing Credentials Securely

**Environment Variables**

```bash title=".env"
API_KEY=your_secret_api_key
CLIENT_ID=your_client_id
CLIENT_SECRET=your_client_secret
```

```javascript title="config.js"
// Good: Load from environment
const apiKey = process.env.API_KEY;

// Bad: Hardcoded credentials
const apiKey = "sk_live_abc123"; // Never do this!
```

### Token Refresh

<Callout type="info">
Access tokens expire after 1 hour. Implement token refresh logic to maintain continuous access.
</Callout>

```javascript title="refresh-token.js"
async function refreshAccessToken(refreshToken) {
  const response = await fetch('https://auth.example.com/oauth/token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      grant_type: 'refresh_token',
      refresh_token: refreshToken,
      client_id: process.env.CLIENT_ID,
      client_secret: process.env.CLIENT_SECRET
    })
  });

  const { access_token, refresh_token: newRefreshToken } = await response.json();
  return { access_token, refresh_token: newRefreshToken };
}
```

## Scopes

OAuth scopes control what your application can access:

| Scope | Description |
|-------|-------------|
| `read:users` | Read user information |
| `write:users` | Create and update users |
| `read:projects` | Read project data |
| `write:projects` | Create and update projects |
| `read:teams` | Read team information |
| `write:teams` | Manage teams |
| `admin` | Full administrative access |

<Callout type="warn">
Request only the scopes you need. Users are more likely to approve requests with minimal permissions.
</Callout>
